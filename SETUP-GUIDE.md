# 🚀 Gemma Avatar — Offline Local Deployment Guide

This repository contains a fully patched, frozen, and ready-to-run local implementation of the **Gemma Avatar** Hugging Face Space ([victor/gemma-avatar](https://huggingface.co/spaces/victor/gemma-avatar)).

---

## 🛠️ Architecture Overview

```text
Windows Browser (Chrome/Edge) @ http://localhost:3000
  │
  ├─ Microphone Audio (PCM16 16kHz base64)
  │     ▼
  ├─ Local WebSocket @ ws://localhost:8765/v1/realtime
  │     │ (Hugging Face speech-to-speech server in WSL2)
  │     ├─ Silero VAD (Voice Activity Detection)
  │     ├─ Parakeet-TDT 1.1B STT
  │     ├─ Ollama Gemma 4 (via http://172.27.81.242:11434/v1 OpenAI Chat API)
  │     └─ Qwen3-TTS 1.7B (CustomVoice - default speaker: Sohee)
  │     ▼
  └─ Returned PCM16 16kHz audio stream + tool calls
        │
        ├─ Audio Worklet -> HeadAudio Viseme Classifier (model-en-mixed.bin)
        ├─ TalkingHead 3D Avatar Stage (brunette.glb)
        └─ Facial Expressions, Lip-Sync, and Gestures
```

---

## 📂 Included Offline Assets & Patches

1. **Local WebSocket Default:** `src/app.js` is patched to default to `ws://localhost:8765/v1/realtime`.
2. **Offline Web Fonts:** External Google Fonts links removed from `index.html`; local `@font-face` definitions provided in `src/style.css` with offline binaries in `public/vendor/`.
3. **Downloaded LFS Objects:** `public/avatars/brunette.glb` (4.72 MB 3D model) and `public/vendor/model-en-mixed.bin` (14.4 KB HeadAudio viseme classifier) are verified binary files.
4. **Frozen Node Dependencies:** Complete `node_modules` generated by `bun install --frozen-lockfile`.
5. **Executable Orchestration Scripts:** `start-ollama.sh`, `start-speech-to-speech.sh`, `start-avatar-frontend.sh`, `run-all-local.sh`, and `test-tools.sh`.

## ⚡ Quick Start (WSL2 / Linux)

### 1. Model Pull (Online Step)
Before disconnecting from the internet, run:
```bash
ollama pull gemma4:31b   # Default local launcher target
# Optional fallback models:
ollama pull gemma4:12b
ollama pull gemma4:26b
```

Pre-download Hugging Face models for speech-to-speech:
```bash
pip install -r gemma-avatar-s2s-requirements-lock.txt
python -c "from huggingface_hub import snapshot_download; snapshot_download('nvidia/parakeet-tdt-1.1b')"
/home/l3ung/venvs/gemma-avatar-s2s/bin/python download_tts_assets.py
```

### 2. Launch Stack Offline
Run all three services:
```bash
./run-all-local.sh
```

Notes:
- The launcher uses `http://127.0.0.1:11434` by default instead of a transient WSL IP.
- It defaults to `gemma4:31b`. If that exact model is unavailable, it falls back to `gemma4:26b`, `gemma4:12b`, then `gemma4:latest`.
- The speech backend assumes a slow `gemma4:31b` warmup, uses a 300 second Ollama-compatible request timeout by default, and defaults Qwen3-TTS to the `torch` backend so no separate GGUF download is required.
- You can override either default with `OLLAMA_MODEL=...` or `LOCAL_LLM_REQUEST_TIMEOUT_S=...`.

Or start individual services in separate terminals:
- **Terminal 1 (LLM):** `./start-ollama.sh`
- **Terminal 2 (Realtime Audio Server):** `./start-speech-to-speech.sh`
- **Terminal 3 (Web UI):** `./start-avatar-frontend.sh`

Open **`http://localhost:3000`** in Chrome or Edge, and click **Start talking**.

## 🧪 Testing & Verification

1. **Synthetic Microphone Test:**  
   Open `http://localhost:3000/?fakemic=1` and run `getClient().requestResponse()` in the browser console.
2. **Tool-Calling Verification:**  
   Run `./test-tools.sh` to verify that Ollama returns `make_hand_gesture` function calls.
3. **Barge-in Test:**  
   Speak while the avatar is talking. Playback clears instantly and mouth movements settle.
